Findings and Conclusions

1. Almost all the search algorithms can always produce a correct shortest path except "depth_first", which can get correct path sometimes but not for sure, depending on the start point and goal point. In addition, "greedy-best-first" seldom fails to find the shortest path, but this can happen in some planning scene with particular start point and goal point. For example, it fails in "misc" with start [0,0] and goal [4,4]

2. If we only take the iteration times into consideration, "depth-first">"greedy-best-first">"A-star">"breadth-first". However, "depth-first" always gives a wrong path, "greedy-best-first" can seldom gives wrong path. "greedy-best-first" achieve less iteration times at the cost of accuracy.

3. Generally speaking, the performance of "A-star" is the best, then followed by "greedy-best-first" and "breadth-first". "depth-first" is the worst one
